## 2026-06-19 Engineering: thêm tương tác real-time qua Cloudflare Worker (webhook Telegram) | Context: khi cần thêm 2-way interaction (bot nhận lệnh/callback) cho 1 pipeline cron 1-chiều hiện có, đặc biệt khi hạ tầng chính (GitHub Actions) không hỗ trợ long-running process để polling

- Pattern kiến trúc: cron pipeline (GitHub Actions, không server) ghép với 1 Cloudflare Worker (free tier, webhook-based, không cần build step — JS thuần + fetch) để có phần tương tác real-time, KHÔNG cần VPS. Pipeline ghi "context" (item đã chọn/verify) lên Cloudflare KV qua REST API ngay sau Stage 2; Worker đọc lại KV khi nhận callback_query/message từ Telegram. State 2 bên (Python pipeline + JS Worker) đồng bộ qua KV, không qua DB nào khác.
- callback_data Telegram giới hạn 64 byte -> dùng index 1-based đơn giản ("qa:1", "qa:2"...) làm key, KHÔNG hash item.id — vừa ngắn vừa dễ khớp giữa code Python ghi KV (`item:<idx>`) và Worker đọc KV, miễn 2 bên dùng đúng cùng thứ tự list.
- Nguyên tắc graceful-degrade áp dụng xuyên cả 2 phía: thiếu secret Cloudflare ở Python -> main.py log warning, bỏ qua ghi KV, KHÔNG sập pipeline cron chính (digest Telegram vẫn gửi, chỉ thiếu nút inline). Thiếu secret ở Worker (GEMINI_API_KEY/GITHUB_PAT/TELEGRAM_BOT_TOKEN) -> Worker luôn answerCallbackQuery trước (tắt loading Telegram) rồi mới xử lý, và luôn sendMessage báo lỗi rõ cho operator thay vì để Telegram timeout im lặng — tuyệt đối không silent-fail ở webhook vì user đang chờ phản hồi tương tác trực tiếp (khác pipeline cron chạy ngầm không ai chờ).
- Vì agent không có tài khoản Cloudflare/GitHub PAT thật, phần deploy/test thật luôn phải handoff cho operator — chỉ viết code + hướng dẫn copy-paste từng bước (wrangler login/kv namespace create/secret put/deploy/setWebhook), kèm hướng dẫn test bằng `wrangler dev` + curl giả update Telegram để operator tự verify trước khi cần Claude Code debug tiếp.
